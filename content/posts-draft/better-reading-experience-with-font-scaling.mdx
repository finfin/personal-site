工程師做久了，上班看桌機、下班看手機，一天盯著螢幕十小時以上是日常。長時間盯著螢幕看，幾年前已經感受到視力明顯的減弱，看小字會開始容易累、眼睛乾。為了閱讀的舒適度，瀏覽器幾乎是常態性的把 zoom 設定在 110% / 125% 、手機則是會把預設字型大小調大。甚至還會把一些排版很差的網頁丟到閱讀器內透過閱讀器來幫助閱讀，比較好控制版面與字型大小。



然後，因為這些設定，就會發現很多奇妙的 bug （笑）



你可能會想問，這樣的使用者有多少？根據統計數據，40 歲以上的人有超過六成有眼睛問題，而 Android 使用者中有超過一成會使用非預設的字型大小。因此，想做出好的網頁瀏覽體驗必須要認真需要考慮這些因素。

# 使用者自訂字型大小的三種常見方式



1. 透過 browser zoom ，這方法是調整像素大小，比如說 zoom 200% 其實就是把一個像素變成 2x2 的大小。桌機使用者透過快速鍵 CMD + / CMD - 就可以放大縮小，就算不知道快速鍵，放大縮小的按鈕也在兩三個點擊就可以到達的地方。

   [螢幕錄影 2024-06-21 上午8.48.50.mov](./螢幕錄影%202024-06-21%20上午8.48.50.mov)

2. 手勢縮放：其實跟瀏覽器 zoom 很像，差別是：zoom 是縮放 pixel 大小，viewport 維持不變，所以整體排版會被影響，因為同樣是 375px 寬度的 zoom 200% 等於顯示寬度變成 167.5px。而手勢縮放則是直接把整個頁面放大 200%。好處是不影響排版，壞處是很多東西會跑到 viewport 之外，如果是想持續閱讀網頁內容的話通常不會用這個方式。

   [螢幕錄影 2024-06-21 上午8.46.11.mov](./螢幕錄影%202024-06-21%20上午8.46.11.mov)

3. 調整預設字型大小：瀏覽器內有一個『字型大小』的選項，到設定下面去找，手機桌機都會有。這其實就是所謂的 rem default，一般來說會是 『正常 - 16px』。但可以調整，桌機 Chrome 的大小是 9 / 12 / 16 / 20 / 24。但因為這只有調整 rem default，所以如果是用 px 定義字形大小的網頁，你就會感受不到調整字型大小的差異，而這種網站佔據大多數。

   [螢幕錄影 2024-06-21 上午8.51.26.mov](./螢幕錄影%202024-06-21%20上午8.51.26.mov)

# 更好的文字閱讀體驗 Font Scaling

這是一種讓排版隨著使用者字型大小偏好而調整的策略。前面提到的三種縮放方法中，前兩種都是無差別的縮放（也最容易實作）。但因為很多時候我們看網頁是在閱讀文字，這時想放大的是文字本身，而不是其他地方如 margin、padding 這些，所以應該要往第三種字型的縮放來作，才能給使用者比較好的體驗。

如果有用過閱讀器或電子書的人應該都有過閱讀器的字型大小的控制，Font Scaling 這個策略其實就是想做到跟電子書一樣，讓使用者可以用自己決定的字型大小舒服的瀏覽網頁。這跟無差別的縮放體驗是完全不同的境界，尤其是在空間比較有限的手機上。

實作上只要注意兩個點：字型＆字型相關的尺寸使用 rem，而字型無關的尺寸如 margin, padding, border 就使用 px。這樣頁面就可以依照使用者偏好的字型大小來作合適的調整。（前提：要先針對不同字型大小確認設計樣式）。

[更詳細的資訊可以參考這篇](https://medium.com/airbnb-engineering/rethinking-text-resizing-on-web-1047b12d2881)

# 幫助除錯

除了更好的閱讀體驗之外，知道這三種縮放方法時，在遇到破版時就有另一種角度可以看：

在桌機上看到平板甚至手機的排版 → 有可能是 zoom 導致斷點變化的問題

手機上的字型大小變化跟你測試的時候不一樣 → 有的地方用 rem 有的地方用 px 設定字型，使用者一調整字型大小時影響到 rem 但 px 沒有跟著變動



# 心得

雖然我自己已經習慣把閱讀體驗差的網頁內容丟進閱讀器來閱讀，但常常還是希望可以在原始的網頁，依據作者的排版來閱讀。這時候一個好的閱讀體驗就是重要的，希望大家可以更了解 Font Scaling 這件事情，幫助我這樣一個眼殘的人的眼睛可以用的久一點。
